DecHW: Decentralized Federated Learning with Second-Order Optimization

A PyTorch-based framework for decentralized federated learning over dynamic peer-to-peer networks, enabling robust training under data and model heterogeneity.

Overview

This project develops a decentralized federated learning framework where clients collaborate over peer-to-peer networks without a central server. Unlike standard federated learning setups, the system supports dynamic communication graphs and neighborhood-based model exchange.

On top of this framework, DecHW introduces a second-order aggregation strategy that leverages curvature information (Hessian-based) to better handle data and model heterogeneity under non-IID conditions, resulting in more stable and efficient distributed training.

System Design

  • Decentralized training architecture with peer-to-peer communication.
  • Support for dynamic graph topologies and neighborhood-based aggregation.
  • PyTorch-based simulation framework for large-scale distributed experiments.
  • Flexible client configuration for heterogeneous data and model settings.

Key Contributions

  • Designed a decentralized federated learning framework without central coordination.
  • Developed DecHW, a second-order aggregation method using curvature information.
  • Improved robustness to data and model heterogeneity in non-IID settings.
  • Demonstrated improved convergence stability compared to standard aggregation methods.

Evaluation

  • Benchmarked against standard federated learning methods (e.g., FedAvg-style aggregation).
  • Evaluated under non-IID data distributions and varying network topologies.
  • Analyzed convergence behavior and communication efficiency.

Tech Stack

Python, PyTorch, Federated Learning, Decentralized Training, Graph-based Communication, Non-IID Learning.
